# Android Media Player

Android Media Player - приложение для локального воспроизведения аудио- и видеофайлов на Android. Пользователь выбирает один файл или несколько файлов через системный выборщик документов, после чего приложение формирует плейлист и воспроизводит медиа через AndroidX Media3 ExoPlayer.

## Возможности

- Открытие одного аудио- или видеофайла через системный `OpenDocument`.
- Открытие нескольких файлов как плейлиста через `OpenMultipleDocuments`.
- Воспроизведение локальных файлов, доступных через URI Android Storage Access Framework.
- Автоматическая подготовка и запуск выбранного медиа.
- Управление воспроизведением: play/pause, перемотка назад и вперед на 10 секунд, переход по позиции через слайдер.
- Отображение видео во встроенной области 16:9.
- Отдельный аудиорежим с визуальной заглушкой для файлов без видеодорожки.
- Полноэкранный режим для видео с сокрытием системных панелей.
- Просмотр информации о текущем файле: имя, тип, статус, позиция и длительность.
- Плейлист с выбором трека и отметкой текущего элемента.
- Отображение ошибок воспроизведения.

## Технологии

| Область | Используется |
| --- | --- |
| Язык | Kotlin |
| UI | Jetpack Compose, Material 3 |
| Медиадвижок | AndroidX Media3 ExoPlayer |
| Интеграция плеера | Media3 `MediaSessionService`, `MediaController`, `PlayerView` |
| Состояние экрана | `AndroidViewModel`, `StateFlow` |
| Сборка | Gradle Kotlin DSL, Android Gradle Plugin |
| Минимальная версия Android | API 24, Android 7.0 |
| Целевая версия Android | API 36 |

```text
mediaplayer/
├── app/

│       │   │   ├── PlaybackService.kt
│       │   │   ├── PlayerUiState.kt
│       │   │   └── PlayerViewModel.kt
│       │   └── res/
│       ├── test/
│       └── androidTest/
├── build.gradle.kts
├── settings.gradle.kts
└── gradle/libs.versions.toml
```

Ключевые классы:

- `MainActivity.kt` - Compose UI, выбор файлов, основной экран, fullscreen-экран и кастомные элементы управления.
- `PlayerViewModel.kt` - подключение к `PlaybackService`, загрузка плейлиста, управление playback-состоянием и синхронизация UI.
- `PlaybackService.kt` - сервис Media3 с экземпляром `ExoPlayer` и `MediaSession`.
- `PlayerUiState.kt` - модели состояния плеера и элемента плейлиста.

## Требования для сборки

- Android Studio с поддержкой Android Gradle Plugin 8.x.
- JDK, совместимый с используемой версией Android Gradle Plugin. Обычно достаточно JDK, встроенного в Android Studio.
- Установленный Android SDK Platform 36.
- Устройство или эмулятор Android API 24+.

## Быстрый старт

Перейдите в корень Android-проекта:

```powershell
cd mediaplayer
```

Соберите debug-версию:

```powershell
.\gradlew.bat :app:assembleDebug
```

Установите debug-версию на подключенное устройство или эмулятор:

```powershell
.\gradlew.bat :app:installDebug
```

Запустите локальные unit-тесты:

```powershell
.\gradlew.bat :app:testDebugUnitTest
```

Запустите инструментальные тесты на устройстве:

```powershell
.\gradlew.bat :app:connectedDebugAndroidTest
```

Для Linux/macOS используйте `./gradlew` вместо `.\gradlew.bat`.

## Использование приложения

1. Запустите `Media Player`.
2. Нажмите `Открыть файл`, чтобы выбрать один аудио- или видеофайл.
3. Нажмите `Открыть плейлист`, чтобы выбрать несколько файлов.
4. Используйте кнопки `Плей`/`Пауза`, `Назад 10с`, `Вперед 10с` и слайдер позиции.
5. Для видео нажмите `Во весь экран`, чтобы перейти в полноэкранный режим.
6. В блоке плейлиста выберите другой трек, если было открыто несколько файлов.

## Поддерживаемые форматы

Приложение передает выбранные URI в ExoPlayer, поэтому набор поддерживаемых форматов зависит от Media3 ExoPlayer, кодеков Android-устройства и контейнера файла. В интерфейсе заявлены популярные форматы: MP3, AAC, WAV, MP4, MKV, WebM и другие форматы, совместимые с ExoPlayer.

## Разрешения

В `AndroidManifest.xml` объявлены:

- `FOREGROUND_SERVICE`
- `FOREGROUND_SERVICE_MEDIA_PLAYBACK`
- `POST_NOTIFICATIONS`

Доступ к файлам выполняется через Storage Access Framework. Приложение не запрашивает широкие разрешения на чтение всей медиатеки, а получает доступ только к выбранным пользователем URI. Для выбранных файлов выполняется попытка сохранить постоянное разрешение на чтение.

## Текущее состояние и ограничения

- Плейлист хранится в состоянии ViewModel и не сохраняется как отдельная сущность после перезапуска приложения.
- Нет поиска по медиатеке, сканирования папок и импорта сетевых потоков.
- Нет режимов shuffle/repeat, эквалайзера, субтитров и отдельного экрана настроек.
- Runtime-запрос разрешения `POST_NOTIFICATIONS` в текущем коде не реализован.
- Unit- и instrumented-тесты сейчас являются шаблонными и не покрывают сценарии плеера.
